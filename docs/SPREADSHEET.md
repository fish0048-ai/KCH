## 1. 分組名單（801A / 804A / 806B）

| 欄位 | 說明 | 範例 |
|------|------|------|
| 自然分組 | 理化分組代號 | `801理A` |
| 班級 | 行政班 | `801`、`802`（可跨班） |
| 座號 | 座號 | `2` |
| 姓名 | 學生姓名 | `蔡昀庭` |
| 加分 | 累計加分 | `2` |

**重點：** 一個分頁 = 一個教學分組（如 `801A名單`），但內含多個行政班學生。

---

## 2. 加分記錄

| 欄位 | 說明 |
|------|------|
| 日期 | `2026/05/28` |
| 時間 | `15:58` |
| 分組 | `806B名單` |
| 班級 | `806` |
| 座號 | `24` |
| 姓名 | `劉辰星` |
| 加分 | `2` |

課堂每次加分都會寫入一筆，與名單上的「加分」累計欄位呼應。

---

## 3. 座位表（JSON 儲存）

| 欄位 | 說明 |
|------|------|
| 分組名稱 | `801A名單` |
| 儲存時間 | `2026/05/29 13:50:37` |
| JSON資料 | 完整座位狀態 |

JSON 內容包含：

```json
{
  "rows": 7,
  "cols": 7,
  "blocked": ["6,0", "6,6"],
  "fixedSeats": { "6,2": { "name": "王章熏", "number": "4", "score1": 76, "score2": 73, "class": "801", "group": "801理A" } },
  "assignments": { "5,1": { "name": "蔡昀庭", "number": "2", "score1": 90, "score2": 91, "class": "801", "group": "801理A" } }
}
```

- `score1` / `score2`：對應第一次、第二次段考（或平時成績來源）
- 每個分組只保留最新一筆有效座位配置（目前 3 筆：801A、804A、806B）

---

## 4. 段考成績分頁

以 `801A第二次段考` 為例：

| 欄位 | 說明 |
|------|------|
| 自然分組 | `801理A` |
| 班級 / 座號 / 姓名 | 學生識別 |
| 3-3~3-4 | 章節成績 |
| 4-1 ~ 4-2 | 章節成績 |
| 5-1 ~5-2 | 章節成績 |
| L3 ~L4 | 章節成績 |
| 標準化平均分 | 統計值 |
| 平時成績 | 顯示在座位表上的成績 |

## 與新系統（Firebase）的對應規劃

| 試算表 | Firestore 集合 |
|--------|----------------|
| 801A/804A/806B名單 | `groups/{groupId}/students` |
| 加分記錄 | `groups/{groupId}/bonusLogs`（待實作） |
| 座位表 JSON | `groups/{groupId}/seating/current` |
| 段考分頁 | `groups/{groupId}/examScores/{examId}`（待實作） |

### 目前已支援
- 從「名單」分頁 CSV 匯入：自然分組、班級、座號、姓名、加分
- 座位表編排、公布學生版

### 待補強

- 匯入「座位表」分頁既有 JSON
- 匯入段考成績並在座位表顯示 `score1`/`score2`
- 加分寫回並同步「加分記錄」
